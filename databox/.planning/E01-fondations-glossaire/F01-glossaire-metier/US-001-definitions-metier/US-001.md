# User Story: Définitions métier fondamentales

## Story

**As a** utilisateur métier interrogeant Databox
**I want** que chaque terme business courant soit traduit en colonnes et filtres SQL précis
**So that** le skill génère du SQL qui reflète fidèlement ce que je mesure dans mon activité

## Acceptance Criteria

### AC1: Terme "Chiffre d'affaires" défini
- **Given** le glossaire métier
- **When** le terme "chiffre d'affaires" est recherché
- **Then** la définition précise quelle table (invoices ou sales_orders), quels filtres (end_date IS NULL, invoices_sign), et quelle colonne de montant utiliser

### AC2: Terme "Client actif" défini
- **Given** le glossaire métier
- **When** le terme "client actif" est recherché
- **Then** la définition inclut le critère temporel (commande ou facture dans les N derniers mois), la table source et les filtres

### AC3: Terme "Devis converti" défini
- **Given** le glossaire métier
- **When** le terme "devis converti" est recherché
- **Then** la définition précise les statuts (quotes_quote_status IN ('2','3')), et si "partiellement commandé" compte comme converti

### AC4: Au moins 10 termes métier validés
- **Given** un dialogue itératif entre Claude et l'utilisateur
- **When** le glossaire est finalisé
- **Then** il contient au moins 10 termes avec pour chacun : définition, table(s), colonne(s), filtre(s) SQL

### AC5: Glossaire stocké dans le format attendu
- **Given** les termes validés
- **When** le glossaire est écrit
- **Then** il est dans references/metier/glossaire.md avec un format structuré (terme → définition → SQL)

## Tasks

1. [ ] Lire DB-SCHEMA.md et identifier les termes métier candidats
2. [ ] Proposer des définitions pour chaque terme, une par une
3. [ ] Valider/ajuster chaque définition avec l'utilisateur
4. [ ] Écrire references/metier/glossaire.md

## Technical Notes

- Termes candidats : CA (facturé vs commandé), marge, client actif/inactif, devis converti, avoir, encours, échéance, taux de conversion, panier moyen, délai de paiement
- Le dialogue doit clarifier les ambiguïtés (ex: CA basé sur factures ou commandes ?)
- Format glossaire : tableau avec colonnes Terme | Définition métier | Tables | Colonnes | Filtres SQL

## Definition of Done

- [ ] Au moins 10 termes validés par l'utilisateur
