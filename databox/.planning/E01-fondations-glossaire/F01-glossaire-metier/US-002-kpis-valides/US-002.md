# User Story: KPIs validés avec formules SQL

## Story

**As a** utilisateur métier
**I want** que chaque KPI ait une formule SQL Databox précise et validée
**So that** les calculs soient reproductibles et cohérents d'une question à l'autre

## Acceptance Criteria

### AC1: KPI "Évolution CA" avec formule SQL
- **Given** la définition de CA du glossaire
- **When** le KPI "évolution CA" est défini
- **Then** il inclut la formule SQL avec generate_series, la granularité (mois/trimestre/année), et le traitement des périodes sans données

### AC2: KPI "Taux de conversion" avec formule SQL
- **Given** la définition de devis converti du glossaire
- **When** le KPI "taux de conversion" est défini
- **Then** il inclut la formule (nb convertis / nb total * 100), les filtres de période, et la possibilité de ventiler par commercial

### AC3: KPI "Marge" avec formule SQL
- **Given** les colonnes de marge dans le schéma (total_margin, line_margin)
- **When** le KPI "marge" est défini
- **Then** il précise la colonne source, le calcul (absolue vs %), et le traitement des avoirs

### AC4: Fichier kpis.md structuré
- **Given** les KPIs validés
- **When** le fichier est écrit
- **Then** chaque KPI a : nom, description, formule SQL complète, paramètres (période, filtres), exemple de résultat attendu

## Tasks

1. [ ] Identifier les KPIs à partir du glossaire et des questions cibles de la spec
2. [ ] Proposer chaque formule SQL et valider avec l'utilisateur
3. [ ] Écrire references/metier/kpis.md

## Technical Notes

- KPIs candidats : évolution CA, taux de conversion, marge brute, panier moyen, top N clients, délai moyen de paiement
- Chaque formule doit appliquer les règles Databox (end_date IS NULL, invoices_sign, aliases id_mapping)

## Definition of Done

- [ ] All acceptance criteria met
- [ ] Au moins 5 KPIs avec formules SQL validées
- [ ] Fichier kpis.md créé
