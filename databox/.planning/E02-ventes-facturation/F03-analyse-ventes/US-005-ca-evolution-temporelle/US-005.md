# User Story: CA et évolution temporelle

## Story

**As a** dirigeant ou responsable commercial
**I want** demander l'évolution de mon chiffre d'affaires sur une période en langage naturel
**So that** je puisse suivre la tendance de mon activité sans écrire de SQL

## Acceptance Criteria

### AC1: CA par mois sur les 12 derniers mois
- **Given** la question "Quelle est l'évolution de mon CA sur les 12 derniers mois ?"
- **When** le skill génère et exécute le SQL
- **Then** le résultat affiche un montant par mois, y compris les mois à zéro, avec generate_series

### AC2: CA par trimestre
- **Given** la question "Quel est mon CA par trimestre cette année ?"
- **When** le skill génère le SQL
- **Then** le résultat agrège par trimestre avec DATE_TRUNC('quarter')

### AC3: Comparaison année N / N-1
- **Given** la question "Compare mon CA de cette année avec l'année dernière"
- **When** le skill génère le SQL
- **Then** le résultat montre les deux totaux et le pourcentage d'évolution

### AC4: CA par commercial
- **Given** la question "Quel est le CA par commercial ce trimestre ?"
- **When** le skill génère le SQL
- **Then** le résultat résout invoices_idcor_sales_rep via id_mapping vers sales_reps

### AC5: Exemples few-shot créés
- **Given** les requêtes validées
- **When** les exemples sont écrits
- **Then** references/exemples/requetes_ventes.md contient au moins 3 paires NL→SQL sur l'évolution CA
