# User Story: Encours clients et échéances

## Story

**As a** responsable financier ou credit manager
**I want** consulter les encours clients et les échéances de paiement
**So that** je puisse suivre le recouvrement et anticiper les retards

## Acceptance Criteria

### AC1: Encours total par client
- **Given** la question "Quel est l'encours de chaque client ?"
- **When** le skill génère le SQL
- **Then** le résultat utilise la table open_items avec résolution du client via idcor

### AC2: Échéances dépassées
- **Given** la question "Quelles échéances sont dépassées ?"
- **When** le skill génère le SQL
- **Then** le résultat filtre customer_due_dates ou open_items sur les dates échues

### AC3: Balance âgée
- **Given** la question "Balance âgée de mes clients"
- **When** le skill génère le SQL
- **Then** le résultat ventile les encours par tranche d'ancienneté (0-30j, 30-60j, 60-90j, >90j)

### AC4: Top clients en retard
- **Given** la question "Quels clients ont le plus de retard de paiement ?"
- **When** le skill génère le SQL
- **Then** le résultat classe les clients par montant échu non payé
