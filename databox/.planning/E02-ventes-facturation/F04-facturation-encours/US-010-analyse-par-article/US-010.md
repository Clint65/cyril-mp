# User Story: Analyse par article/famille sur le cycle vente

## Story

**As a** responsable commercial ou product manager
**I want** analyser les ventes par article ou famille de produits
**So that** je puisse identifier les produits performants et ajuster mon offre

## Acceptance Criteria

### AC1: Top articles par CA
- **Given** la question "Quels sont mes 10 articles les plus vendus en CA ?"
- **When** le skill génère le SQL
- **Then** le résultat joint les lignes de facture aux articles via id_product::text avec le cast correct

### AC2: Top articles par quantité
- **Given** la question "Quels articles se vendent le plus en volume ?"
- **When** le skill génère le SQL
- **Then** le résultat agrège les quantités depuis sales_orders_lines ou invoices_lines

### AC3: Analyse par famille
- **Given** la question "CA par famille de produits cette année"
- **When** le skill génère le SQL
- **Then** le résultat utilise articles_family_code ou les colonnes family_code des lignes

### AC4: Cycle complet d'un article
- **Given** la question "Résumé de l'activité de l'article X (devis, commandes, factures)"
- **When** le skill génère le SQL
- **Then** le résultat utilise le pattern UNION ALL du schéma (pattern 4.12) sur quotes_lines, sales_orders_lines, invoices_lines
