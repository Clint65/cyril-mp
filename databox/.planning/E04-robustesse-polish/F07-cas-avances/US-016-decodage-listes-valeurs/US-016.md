# User Story: Décodage listes de valeurs

## Story

**As a** utilisateur métier
**I want** que les codes (statuts, types, catégories) soient affichés en libellés lisibles
**So that** les résultats soient compréhensibles sans connaître les codes internes

## Acceptance Criteria

### AC1: Décodage statut devis
- **Given** la question "Liste des devis avec leur statut"
- **When** le skill génère le SQL
- **Then** le résultat affiche "Ouvert", "Commandé", "Partiellement commandé" au lieu de '1', '2', '3'

### AC2: Décodage type de compte
- **Given** la question "Répartition des clients par type de compte"
- **When** le skill génère le SQL
- **Then** le résultat utilise le pattern value_list_column_config → value_list_entry → value_list_label

### AC3: Découverte dynamique des colonnes décodables
- **Given** une colonne codée inconnue
- **When** le skill cherche si elle a un décodage
- **Then** il consulte value_list_column_config pour trouver le value_list_id correspondant

### AC4: Décodage en français
- **Given** le pattern de décodage
- **When** le label est récupéré
- **Then** le filtre locale = 'fr' est appliqué sur value_list_label

- column_name dans value_list_column_config est en camelCase (ex: 'accountsAccountType')
