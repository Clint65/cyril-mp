# User Story: Créer le SKILL.md final via /skill-creator

## Story

**As a** utilisateur métier
**I want** un skill Claude Code opérationnel assemblant toutes les références construites dans E01-E04
**So that** je puisse interroger Databox en langage naturel avec un seul point d'entrée fiable

## Acceptance Criteria

### AC1: SKILL.md créé via /skill-creator
- **Given** le glossaire (E01/F01), les KPIs (E01/F01), les exemples few-shot (E01/F02, E02, E03, E04/F08), les pièges (E01/F02) et la checklist de validation (E04/F08)
- **When** /skill-creator est invoqué
- **Then** un SKILL.md est généré intégrant toutes ces références comme contexte

### AC2: Logique d'exécution en 5 étapes intégrée
- **Given** le SKILL.md
- **When** une question est posée
- **Then** le skill suit les 5 étapes : COMPRENDRE → IDENTIFIER → GÉNÉRER → VALIDER → EXÉCUTER

### AC3: Références dynamiques vers les fichiers construits
- **Given** le SKILL.md
- **When** il est lu
- **Then** il référence les fichiers du dossier references/ (glossaire.md, kpis.md, requetes_ventes.md, requetes_achats.md, requetes_fondamentales.md, pieges_connus.md)

### AC4: Les 6 règles critiques dans l'étape VALIDER
- **Given** le SKILL.md
- **When** l'étape VALIDER est consultée
- **Then** elle contient la checklist des 6 règles (end_date IS NULL, idcor, aliases, cast TEXT, invoices_sign, value_lists)

### AC5: Test end-to-end sur les 4 questions cibles
- **Given** le SKILL.md opérationnel
- **When** les 4 questions de la spec sont posées ("évolution CA 12 mois", "taux de conversion par commercial", "clients inactifs", "top 10 clients par CA")
- **Then** chaque question produit une réponse correcte en langage naturel avec données

## Tasks

1. [ ] Consolider la liste de tous les fichiers references/ produits par E01-E04
2. [ ] Invoquer /skill-creator avec le contexte complet (spec, glossaire, KPIs, exemples, pièges, checklist)
3. [ ] Valider le SKILL.md produit sur les 4 questions cibles de la spec
4. [ ] Ajuster si nécessaire

## Technical Notes

- /skill-creator doit recevoir en contexte : skill-nl-sql-erp-crm.md (spec), DB-SCHEMA.md, ID-MAPPING.md, VALUE-LISTS-ARCHITECTURE.md + tous les fichiers references/
- Le SKILL.md doit utiliser le serveur MCP PostgreSQL configuré dans .mcp.json
- L'outil mcp__databox-db__query est le seul outil SQL disponible

## Definition of Done

- [ ] All acceptance criteria met
- [ ] SKILL.md créé via /skill-creator
- [ ] 4 questions cibles testées avec succès
- [ ] Le skill est invocable et opérationnel
