# User Story: Exemples few-shot complets (15-20 paires)

## Story

**As a** skill NL→SQL
**I want** disposer de 15-20 paires NL→SQL validées couvrant tous les domaines
**So that** je puisse m'appuyer sur une base d'exemples complète pour toute question métier

## Acceptance Criteria

### AC1: Couverture des domaines
- **Given** les exemples few-shot
- **When** la couverture est vérifiée
- **Then** tous les domaines sont couverts : ventes (5+), facturation (3+), achats (2+), production (2+), projets (1+), cas avancés (2+)

### AC2: Couverture des patterns SQL
- **Given** les exemples few-shot
- **When** les patterns sont vérifiés
- **Then** tous les patterns critiques sont représentés : version courante, idcor, cast TEXT, generate_series, invoices_sign, value_lists, multi-devises

### AC3: Validation par exécution
- **Given** les 15-20 exemples
- **When** chaque requête est exécutée sur la base
- **Then** aucune erreur SQL et résultats cohérents

### AC4: Organisation par fichier
- **Given** les exemples validés
- **When** ils sont organisés
- **Then** ils sont répartis dans requetes_ventes.md, requetes_achats.md, requetes_fondamentales.md

## Tasks

1. [ ] Consolider les exemples créés dans les stories précédentes
2. [ ] Compléter pour atteindre 15-20 paires
3. [ ] Valider chaque exemple par exécution
4. [ ] Organiser dans les fichiers references/exemples/

## Technical Notes

- Les exemples des stories E02/E03 constituent la base
- Compléter avec des questions variantes et des cas edge

## Definition of Done

- [ ] All acceptance criteria met
- [ ] 15-20 exemples validés par exécution
- [ ] Fichiers exemples organisés par domaine
