# User Story: Validation et tests de non-régression

## Story

**As a** mainteneur du skill
**I want** un jeu de tests vérifiant l'application systématique des 6 règles critiques Databox
**So that** toute modification du skill ne puisse pas introduire de régressions silencieuses

## Acceptance Criteria

### AC1: Test règle end_date IS NULL
- **Given** un jeu de requêtes de test
- **When** chaque requête est analysée
- **Then** toute requête sur une table versionnée contient le filtre end_date IS NULL via id_mapping

### AC2: Test règle aliases id_mapping
- **Given** une requête avec plusieurs jointures id_mapping
- **When** la requête est analysée
- **Then** chaque jointure a un alias distinct

### AC3: Test règle cast TEXT
- **Given** une requête joignant des articles via id_product
- **When** la requête est analysée
- **Then** le cast ::text est présent

### AC4: Test règle invoices_sign
- **Given** une requête d'agrégation sur invoices
- **When** la requête est analysée
- **Then** invoices_sign est utilisé dans le SUM

### AC5: Checklist de validation documentée
- **Given** les 6 règles critiques
- **When** la checklist est finalisée
- **Then** elle est exploitable comme vérification systématique avant exécution de toute requête

## Tasks

1. [ ] Créer la checklist de validation des 6 règles
2. [ ] Créer des requêtes-test pour chaque règle
3. [ ] Documenter dans references/exemples/pieges_connus.md

## Technical Notes

- Les 6 règles : end_date IS NULL, résolution idcor, cast TEXT, aliases id_mapping, invoices_sign, décodage value_lists
- La checklist sera intégrée dans le SKILL.md à l'étape VALIDER (étape 4 de la logique d'exécution)

## Definition of Done

- [ ] All acceptance criteria met
- [ ] Checklist de validation créée
- [ ] Requêtes-test exécutées avec succès
